Lançada versão 0.38.0 da Biblioteca Spartacus   

O desenvolvimento da Spartacus está tomando proporções cada vez maiores. Já ultrapassamos 160 mil linhas de código. Você poderá fazer download da Spartacus 0.38.0 clicando nesse link.

Para ajudar a organizar o desenvolvimento e a correção de bugs, criamos um Trello Board público, onde você poderá ver quais sáo alguns de meus planos para essa biblioteca, e também acompanhar o progresso de cada tarefa.

Toda a documentaço das classes agora poder ser acessada online através desse link. Também comecei a centralizar os artigos e manuais explicativos da Spartacus na wiki do projeto no GitHub.

Segue lista de mudanças:

  • Spartacus.Net.Cryptor:
    • Implementada função para obter strings aleatórias para diversos propósitos.

  • Spartacus.Database:
    • Suporte ao Mono 4.0;
    • Sobrecarga para a função Query, que agora consegue buscar dados em blocos. Isso reduz o uso de memória quando há muitos dados para trazer do banco.

  • Spartacus.Utils.Excel:
    • Suporte a mais fórmulas;
    • Tabelas dinâmicas agora podem ser agrupadas por mais de uma coluna;
    • Suporte à opção "Quebrar texto automaticamente";
    • Suporte a comentários nas células;
    • Formatação condicional para linhas baseado no valor de uma coluna;
    • Incorporado código-fonte da EPPlus 4.0.4 para corrigir um bug NullReferenceException na classe ExcelRangeBase.

  • Spartacus.Reporting:
    • Suporte a bordas no cabeçalho e rodapé do relatório;
    • Objetos de texto agora podem ser centralizados;
    • Caracteres acentuados agora sáo renderizados corretamente. Se os dados do banco vierem com HTML Encoding, a conversáo é automática;
    • Incorporado código-fonte da PDFjet 5.28, e com isso nós:
      • Removemos a dependência da Windows Forms para cortar o texto para caber na célula;
      • Reduzido uso de memória em aproximadamente 50% para renderizar relatórios em PDF;
      • Reduzido tempo de processamento em aproximadamente 75% para renderizar relatórios em PDF;
      • Agora um relatório com aproximadamente 350 mil registros e 10 mil páginas é renderizado em menos de 15 minutos em uma máquina modesta de 32 bits, um único processador e 3 GB RAM.

  • E muito mais otimizações e correções de bugs.

Curitiba, 6 de maio de 2015.

Lançada versão 0.35.2 da Biblioteca Spartacus   

Estou disponibilizando para download a Spartacus 0.35.2, bem como sua documentação.

Mais uma vez, diversas novas funcionalidades foram implementadas e vários bugs foram corrigidos. Tenho novamente o orgulho de disponibilizar a Spartacus aqui no site, para quem não está acompanhando o projeto no Github, onde publico quase toda semana.

Segue lista das principais mudanças desde a versão 0.32.0:

  • Implementada funcionalidade de exportar planilhas do XLSX a partir de um DataTable e um arquivo template XLSX existente;
  • Novos componentes para Spartacus.Forms: Passwordbox, Memobox, Datetimepicker e Filepicker.
  • Na Spartacus.Reporting, foi corrigido um bug em que o texto ficava "encavalado" quando era maior do que a coluna, e foi otimizado o tempo de renderização, pois agora busca a imagem do cabeçlho das páginas apenas uma vez, replicando para as demais páginas;
  • Também na Spartacus.Reporting, foi implementada a funcionalidade de calcular os totalizadores dos grupos automaticamente, sem precisar passar esses valores na consulta;
  • Implementados "pacotes" de relatórios Excel e PDF;
  • Spartacus.Utils.Excel agora consegue gerar arquivos XLSX sem necessidade de template, usando apenas o DataTable;
  • Geração de relatórios em Excel e PDF agora possuem evento de progresso, em que é possível acompanhar o percentual de renderização de cada relatório.

Curitiba, 18 de fevereiro de 2015.

Lançada versão 0.32.0 da Biblioteca Spartacus   

Estou disponibilizando para download a Spartacus 0.32.0, bem como sua documentação.

Muita coisa mudou desde a última versão que eu disponibilizei aqui no site, 0.30.0. Note que a versão 0.31.0 não chegou a ser publicada aqui. O motivo disso é que a versão 0.31.0 foi utilizada em produção, mas foram descobertos alguns problemas e eu não julguei estável o suficiente para disponibilizar para vocês.

Segue lista de mudanças:

  • Removida dependência de Mono.Security.dll, que causava problemas ao utilizar WebClient;
  • Implementada funcionalidade BulkInsert para Spartacus.Database, para inserir dados em massa em bancos de dados;
  • Adicionada classe Spartacus.Utils.ErrorEvent, para tratar eventos de erro em seu aplicativo;
  • Exceções padronizadas e agora mostram stacktrace, para facilitar a depuração do seu código;
  • Removida dependência de ICSharpCode.SharpZipLib, cuja licença não era boa para projetos comerciais. Agora foi incorporado o código da ZipStorer, que permite compactar e descompactar arquivos zip;
  • Removida dependência de EPPlus. Ao invés disso, foi incorporado o código da SejExcel, que consegue ler e escrever arquivos XLSX 100 vezes mais rápido do que a EPPlus (é verdade, eu testei!);
  • Criado namespace Spartacus.ThirdParty, para armazenar códigos de bibliotecas de terceiros;
  • Mesclada classe Spartacus.Utils.CSVTable dentro da classe Spartacus.Utils.Excel;
  • Diversas correções de bugs e otimizações de uso de memória.

Curitiba, 23 de outubro de 2014.

Biblioteca Spartacus   

É com muita satisfação que eu informo a todos que publiquei a minha biblioteca Spartacus no GitHub!!

Para quem não sabe, o GitHub é o maior repositório de projetos de código aberto do mundo. Hospeda o código-fonte de projetos importantes, como o Mono, o JQuery e até mesmo o kernel do Linux, que pertence a Linus Torvalds, o criador do Linux.

A Spartacus é uma biblioteca em C# que venho desenvolvendo há quase 4 anos. Comecei a desenvolver em casa por hobbie, quando percebi que estava tendo que escrever muito código repetido para tarefas semelhantes no meu dia-a-dia. O código todo já chegou a ter mais de 20 mil linhas, e então eu separei em duas partes: criei uma segunda camada somente com código de manipulação de arquivos NF-e, SPED e outras expertises do meu emprego. Essa segunda camada não é pública, pertence ao meu contratante.

A primeira camada da Spartacus, que estou abrindo sob a licença MIT, tem cerca de 12 mil linhas de código, e se beneficia de algumas outras bibliotecas também de código aberto. Abrir a Spartacus foi a forma que eu encontrei de retribuir à comunidade e respeitar suas licenças e seus créditos.

Por enquanto, a Spartacus possui os seguintes módulos:

  • Spartacus.Database: Contém encapsulamentos para acessar diversos bancos de dados diferentes de forma semelhante. Exemplos de SGBDs suportados: Firebird, PostgreSQL, MySQL, SQLite, Oracle (via ODBC) e muitos outros via ODBC e OLEDB.
  • Spartacus.Forms: Permite o desenvolvimento rápido de formulários e telas simples, sem a necessidade de usar designer de interfaces. Os componentes foram projetados para serem bem fáceis de configurar.
  • Spartacus.Net: Suporte para desenvolver aplicativos que se comunicam pela rede. Esquema cliente e servidor e definição de pacotes de dados.
  • Spartacus.Reporting: PDF Report Engine, permite renderizar relatórios em PDF a partir de um arquivo XML que contém a definição do layout do relatório e dados extraídos com o módulo Spartacus.Database.
  • Spartacus.Utils: Classes para extração de dados de arquivos do Microsoft Excel (CSV, XLS e XLSX) e outras coisas.

O projeto já está na versão 0.30.0, que em termos de comentários está completamente documentada. Ainda preciso escrever um wiki no GitHub, elaborar manuais de utilização, definição do layout XML para geração de relatórios em PDF... Enfim, o projeto está longe de ficar pronto, pelo contrário, agora é que está começando.

Criei uma página específica para a Spartacus aqui no site, e disponibilizarei aqui mesmo os binários compilados para serem baixados gratuitamente. Você até já pode baixar a versão 0.30.0!!

Bom, o fato é que a Spartacus agora está disponível no GitHub. O código-fonte é aberto sob a licença MIT, o que significa que você pode baixar e usar em seu aplicativo comercial. Esse é o primeiro projeto de código aberto de minha autoria, e ainda estou aprendendo como esse mundo fascinante funciona!

Curitiba, 11 de setembro de 2014.
